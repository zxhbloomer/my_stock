## 1. 背景与目标

当前项目中回测报告逻辑散落在各个脚本里，每个脚本自己画图，无法复用。需要将报告生成能力抽成独立模块，实现"策略脚本只管回测，报告模块统一出图"的分层架构。

**目标：**
- 任何策略跑完后，喂数据进去就能生成中文 HTML 报告
- 报告样式统一，改一处全部生效
- 参考 czsc 三层架构，不依赖 czsc 库

---

## 2. 架构设计

## 3. 模块详细设计

### 3.1 `utils/plotting/backtest_charts.py`

5个独立绘图函数，每个返回 Plotly Figure 对象：

| 函数名 | 图表 | 输入 |
|--------|------|------|
| `plot_cumulative_returns(returns_df)` | 累计收益曲线（策略 vs 买持） | DataFrame，列为策略名 |
| `plot_drawdown_analysis(returns_series)` | 回撤水下图 + 累计收益双轴 | Series |
| `plot_monthly_heatmap(returns_series)` | 月度收益热力图（年×月） | Series |
| `plot_return_distribution(returns_series)` | 收益分布直方图 | Series |
| `plot_price_bbi_signals(price_df, trades_df)` | K线 + BBI线 + 买卖点 | DataFrame |

所有函数：
- 标题、坐标轴、图例全部中文
- 支持 `to_html=True` 参数返回 HTML div 字符串
- 颜色常量统一定义在文件顶部

### 3.2 `utils/report_builder.py`

通用 HTML 构建器，链式调用：

```python
class BacktestReportBuilder:
    def __init__(self, title, subtitle, theme='light')
    def add_metrics(self, metrics: list[dict]) -> self   # 指标卡
    def add_chart(self, fig, title) -> self              # 添加图表
    def save(self, output_path: str) -> str              # 保存HTML
```

内置 CSS：
- 指标卡（绿色正收益/红色负收益）
- 图表卡（带标题栏）
- 响应式布局（max-width: 96%）
- 浅色主题（可扩展深色）

### 3.3 `scripts/50_个股回测/53_BBI回测_中文报告.py`

业务脚本，职责：
1. 从 PostgreSQL 读数据
2. 用 backtrader 跑 BBI 回测
3. 整理数据（收益率、交易记录、指标）
4. 调用 `BacktestReportBuilder` 生成 HTML

```python
# 调用示例
from utils.report_builder import BacktestReportBuilder
from utils.plotting.backtest_charts import (
    plot_cumulative_returns, plot_drawdown_analysis,
    plot_monthly_heatmap, plot_return_distribution,
    plot_price_bbi_signals
)

builder = BacktestReportBuilder(
    title=f"BBI策略回测报告 — {name}（{code}）",
    subtitle=f"{TRADE_START} ~ {END_DATE} · 周线 · 初始{INIT_CASH/10000:.0f}万元"
)
builder.add_metrics([
    {"label": "总收益", "value": f"{total_return:.1%}", "is_positive": total_return > 0},
    {"label": "年化收益", "value": f"{annual_return:.1%}", "is_positive": annual_return > 0},
    {"label": "最大回撤", "value": f"{max_dd:.1%}", "is_positive": max_dd > -0.2},
    {"label": "夏普比率", "value": f"{sharpe:.2f}", "is_positive": sharpe > 1},
    {"label": "交易次数", "value": f"{trade_count}次", "is_positive": True},
    {"label": "胜率", "value": f"{win_rate:.1%}" if win_rate else "N/A", "is_positive": win_rate and win_rate > 0.5},
])
builder.add_chart(plot_cumulative_returns(returns_df), "累计收益曲线")
builder.add_chart(plot_drawdown_analysis(strategy_returns), "回撤分析")
builder.add_chart(plot_monthly_heatmap(strategy_returns), "月度收益热力图")
builder.add_chart(plot_return_distribution(strategy_returns), "收益分布")
builder.add_chart(plot_price_bbi_signals(price_df, trades_df), "价格与BBI信号")
builder.save(f"scripts/50_个股回测/reports/BBI回测_{name}_{code[:6]}.html")
```

## 4. 数据接口规范

### 收益率 Series
```
index: DatetimeIndex（周频）
values: float（小数，如 0.03 表示 3%）
name: 策略名称（如 "BBI策略"）
```

### 交易记录 DataFrame
```
columns: date, action, price, shares
action: "买入" | "卖出"
```

### 指标卡 list[dict]
```python
{"label": str, "value": str, "is_positive": bool}
```

## 5. 技术约束

- Python 3.11（`mystock311` 环境）

# v4_plan_1 卖出逻辑增强 — 设计文档

**日期**: 2026-05-01  
**基准版本**: v4_plan（BBI 周度轮动策略）  
**目标版本**: v4_plan_1（增加多维卖出逻辑）

---

## 1. 需求背景

v4_plan 是一个 BBI 周度轮动策略，每周一选 Top 5 涨幅股换仓。  
当前卖出逻辑只有一种：**不在 Top N 就卖**（周一换仓时被动卖出）。

问题：本金只有 50 万，没有任何止损保护，一旦持仓大幅下跌，资金会被严重侵蚀，影响后续买入能力。

目标：从 v3 移植完整的卖出逻辑，在轮动框架内实现**逐日止损检查**。

---

## 2. 卖出逻辑设计（5 种，按优先级）

| 优先级 | 名称 | 触发条件 | 说明 |
|--------|------|----------|------|
| 1 | 硬止损 | 亏损 ≥ 8% | 任意交易日，立即卖出，不受最小持仓天数限制 |
| 2 | ATR 追踪止损 | 收盘价 < peak_close - 4.5×ATR14 | 每日更新峰值，跌破追踪线卖出 |
| 3 | BBI 死叉 | 前日收盘 > BBI，当日收盘 < BBI | 价格从上方跌破 BBI |
| 4 | MACD 死叉 | MACD < Signal 且 MACD < 0 且 BBI 3日下行 | 需同时满足 3 个条件，避免假信号 |
| 5 | 筹码胜率过高 | winner_rate（T-1）> 80 | 获利盘过多，卖压大 |

**最小持仓保护**：条件 3/4/5 需持仓 ≥ 20 日才生效（避免刚买入就被噪音信号卖出）。  
条件 1/2（止损类）不受此限制，任何时候都可触发。

## 3. 架构变更

### 3.1 数据层（10_prepare_data.py）

需新增预计算列：

| 列名 | 计算方式 | 说明 |
|------|----------|------|
| `atr14` | Wilder ATR(14) | 每日 ATR，用于追踪止损 |
| `macd` | EMA(12) - EMA(26) | MACD 线 |
| `macd_signal` | EMA(macd, 9) | 信号线 |

这些是纯技术指标，无未来数据泄露风险，不需要 shift。

### 3.2 回测层（20_run_backtest.py）

**新增持仓状态字典**，每只持仓维护：
```python
holdings[code] = {
    'shares': int,
    'cost_price': float,
    'buy_date': str,
    'name': str,
    'peak_close': float,   # 新增：追踪止损用
    'trail_stop': float,   # 新增：当前追踪止损线
    'hold_days': int,      # 新增：持仓天数计数
}
```

**回测主循环修改**：

```
for date in all_dates:
    # 1. 逐日检查止损（非周一也执行）
    for code in list(holdings.keys()):
        if should_stop_loss(code, date, data, holdings):
            执行卖出，cash += 回收资金
            del holdings[code]
            记录 trade_record（标注卖出原因）
    
    # 2. 周一换仓（原有逻辑）
    if date in monday_set:
        选股 → 卖出不在 Top N 的 → 买入新进的
    
    # 3. 日净值计算（原有逻辑）
    计算 nav_series
```

**should_stop_loss 函数**：
```python
def should_stop_loss(code, date, data, holdings):
    pos = holdings[code]
    df = data[code]
    row = df[df['trade_date'] == date]
    if row.empty:
        return False
    
    close = row['close_qfq'].iloc[0]
    cost = pos['cost_price']
    hold_days = pos['hold_days']
    
    # 更新追踪止损
    if close > pos['peak_close']:
        pos['peak_close'] = close
    atr = row['atr14'].iloc[0]
    if pd.notna(atr) and atr > 0:
        new_stop = pos['peak_close'] - 4.5 * atr
        pos['trail_stop'] = max(pos.get('trail_stop', 0), new_stop)
    
    # 1. 硬止损（无持仓天数限制）
    if (close - cost) / cost <= -0.08:
        return True, '硬止损'
    
    # 2. ATR 追踪止损（无持仓天数限制）
    if pos['trail_stop'] and close < pos['trail_stop']:
        return True, 'ATR追踪止损'
    
    # 3-5. 信号类止损（需持仓 >= 20 日）
    if hold_days < 20:
        return False, None
    
    # 3. BBI 死叉
    prev_row = df[df['trade_date'] < date].tail(1)
    if not prev_row.empty:
        prev_close = prev_row['close_qfq'].iloc[0]
        prev_bbi = prev_row['bbi_qfq'].iloc[0]
        if prev_close > prev_bbi and close < row['bbi_qfq'].iloc[0]:
            return True, 'BBI死叉'
    
    # 4. MACD 死叉 + BBI 下行
    macd = row['macd'].iloc[0]
    signal = row['macd_signal'].iloc[0]
    bbi_now = row['bbi_qfq'].iloc[0]
    bbi_3d_ago = df[df['trade_date'] < date].tail(3)['bbi_qfq'].iloc[0] if len(df[df['trade_date'] < date]) >= 3 else None
    bbi_declining = bbi_3d_ago is not None and bbi_now < bbi_3d_ago
    if pd.notna(macd) and pd.notna(signal):
        if macd < signal and macd < 0 and bbi_declining:
            return True, 'MACD死叉'
    
    # 5. 筹码胜率
    wr = row['winner_rate'].iloc[0]
    if pd.notna(wr) and wr > 80:
        return True, '筹码胜率过高'
    
    return False, None
```

### 3.3 资金管理（关键）

止损卖出后，现金立即回流到 `cash`。  
**当天不买入**，等下周一换仓时，`cash` 已包含回收资金，`alloc` 计算自然覆盖。

原有买入分配逻辑不变：
```python
alloc = min(cash / max(len(new_set - cur_set), 1), INIT_CASH / TOP_N * 1.2)
```

## 5. KISS 原则 7 问回答

1. **真问题**：是。v4_plan 无止损，50万本金在熊市会被大幅侵蚀。
2. **更简单的方法**：只加硬止损最简，但用户选择了全部移植，接受复杂度。
3. **会破坏什么**：不会。v4_plan 不动，v4_plan_1 是独立目录。
4. **真的需要**：是。对比不同卖出逻辑收益是明确需求。
5. **过度设计**：否。5种卖出条件都来自 v3 已验证逻辑，无新发明。
6. **话题模糊**：否。需求明确，代码可直接参考。
7. **代码实施注意事项**：已遵守 T-1 shift 规则（winner_rate 已在 prepare_data 中 shift，ATR/MACD 无需 shift）。

---

## 6. 风险

- ATR 计算：Wilder ATR 与简单 ATR 有差异，使用 pandas 的 `ewm` 实现 Wilder 平滑。
- MACD 计算：使用 `ewm(span=N, adjust=False)` 与 backtrader 结果一致。
- 逐日循环性能：每日对每只持仓查 DataFrame，数据量大时可能慢，可用 dict 索引优化。

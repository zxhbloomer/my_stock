# v4_plan_1 交易跟踪系统 — 设计文档

日期：2026-05-02

## 背景

v4_plan_1 是 BBI 周度轮动策略的回测系统，每周五生成"下周操作计划"（买入/卖出哪些股票）。
目前计划只在报表 HTML 里展示，无法保存、无法录入实际成交、无法跟踪执行情况。

本设计新增：
1. 报表页"下周操作计划"区域显示当前本金，并支持一键保存计划到数据库
2. 独立的交易跟踪页面，支持查看历史计划、录入实际成交、跟踪执行状态

---

## 整体架构

```
30_generate_report.py  (静态 HTML, 端口 8085)
    └── 新增"生成计划"按钮 → POST http://localhost:8086/api/plans

40_trading_server.py   (Flask, 端口 8086)
    ├── GET  /                    → trading_tracker.html
    ├── GET  /api/capital         → 查询当前可用本金
    ├── POST /api/plans           → 保存计划（主表 + 明细）
    ├── GET  /api/plans           → 查询计划列表
    ├── GET  /api/plans/<id>/items → 查询某计划的明细
    ├── POST /api/plans/<id>/items/<item_id>/confirm → 确认成交
    └── POST /api/plans/<id>/items/<item_id>/skip    → 跳过

数据库：PostgreSQL my_stock，schema: tushare_v2
    ├── trading_plans       (计划主表)
    └── trading_plan_items  (计划明细)
```

## 数据库表结构

### trading_plans（计划主表）

每次点击"生成下周操作计划"创建一条记录。

```sql
CREATE TABLE tushare_v2.trading_plans (
    id          SERIAL PRIMARY KEY,
    plan_date   DATE        NOT NULL,           -- 计划生成日期（周五收盘后）
    exec_date   DATE,                           -- 预计执行日期（下周一）
    capital     NUMERIC(12,2) NOT NULL,         -- 生成时的可用本金
    status      VARCHAR(10)  DEFAULT 'pending', -- pending / partial / done
    created_at  TIMESTAMP    DEFAULT CURRENT_TIMESTAMP
);
```

### trading_plan_items（计划明细）

每只股票一行，包含计划数据和实际成交数据。

```sql
CREATE TABLE tushare_v2.trading_plan_items (
    id             SERIAL PRIMARY KEY,
    plan_id        INTEGER      NOT NULL REFERENCES tushare_v2.trading_plans(id),
    action         VARCHAR(4)   NOT NULL,   -- 买入 / 卖出
    ts_code        VARCHAR(15)  NOT NULL,
    name           VARCHAR(20),
    ref_price      NUMERIC(10,3),           -- 参考价（上周五收盘价）
    plan_shares    INTEGER,                 -- 计划股数
    plan_amount    NUMERIC(12,2),           -- 计划金额（ref_price × plan_shares）
    ret5           NUMERIC(8,4),            -- 5日涨幅（选股依据）
    remark         VARCHAR(20),             -- 新建仓 / 继续持有 / 清仓
    -- 实际成交（确认后填入）
    actual_price   NUMERIC(10,3),           -- 实际成交价（开盘价）
    actual_shares  INTEGER,                 -- 实际成交股数
    actual_amount  NUMERIC(12,2),           -- 实际金额（actual_price × actual_shares，自动计算）
    confirmed_at   TIMESTAMP,               -- 确认时间
    status         VARCHAR(10)  DEFAULT 'pending'  -- pending / done / skip
);

CREATE INDEX ON tushare_v2.trading_plan_items (plan_id);
```

## 本金计算逻辑

"当前本金"= 用户在 config.py 里手动维护的 `REAL_CASH`，含义是**券商账户当前可用资金**（不含持仓市值）。

```python
# config.py 新增
REAL_CASH = 500_000.0  # 每次换仓后，从券商 App 查看可用资金并更新此值
```

**为什么不从数据库推算：**
- 轮动策略始终接近全仓，现金余额接近 0，可用资金主要来自本次卖出回收
- 卖出 T+1 结算、手续费滑点、止损等因素导致推算误差累积
- 用户直接从券商 App 读取可用资金最准确，一个数字，无歧义

**买入金额分配：**
```python
alloc_per_stock = REAL_CASH / TOP_N  # 与回测逻辑一致，按 TOP_N 平均分配
```

**报表页显示：** 直接读取 config.REAL_CASH，不需要 API 查询。
Flask 的 `GET /api/capital` 接口返回此值供前端展示。

## 报表页改动（30_generate_report.py）

### 改动点 1：显示当前本金

在"下周操作计划"section 标题旁，通过 JavaScript 异步请求 `GET http://localhost:8086/api/capital`，
将返回的本金金额渲染到标题行右侧：

```
下周操作计划（基于 2026-05-02 数据）          当前本金：¥ 498,000
```

若 Flask 服务未启动（请求失败），静默不显示，不影响报表其他功能。

### 改动点 2：生成计划按钮

在操作计划表格下方新增按钮：

```html
<button onclick="savePlan()">生成下周操作计划</button>
```

点击后：
1. 从页面内嵌的 JSON 数据中读取 buy_plan / sell_plan
2. POST 到 `http://localhost:8086/api/plans`
3. 成功后显示提示："计划已保存 → 前往交易跟踪页面"（链接到 http://localhost:8086/）
4. 若重复点击（同一 plan_date 已存在），服务端返回 409，前端提示"本周计划已存在"

## 交易跟踪页面（trading_tracker.html）

### 页签

顶部两个页签，点击"回测报表"跳转到 `http://localhost:8085/report.html`。

### 计划列表

| 计划日期 | 执行日期 | 本金 | 股票数 | 状态 | 操作 |
|---------|---------|------|-------|------|------|
| 2026-05-02 | 2026-05-05 | ¥498,000 | 5只 | 待执行 | [展开▼] |

状态说明：
- `pending`：所有 items 均未确认
- `partial`：部分 items 已确认
- `done`：所有 items 均已确认或跳过

### 行内展开明细

点击"展开"后，该行下方插入子表格：

| 操作 | 代码 | 名称 | 参考价 | 计划股数 | 计划金额 | 5日涨幅 | 状态 | 操作 |
|------|------|------|-------|---------|---------|--------|------|------|
| 买入 | 000001 | 平安银行 | 12.500 | 7900 | 98,750 | +8.3% | 待确认 | [确认成交] |
| 卖出 | 000049 | 德赛电池 | - | - | - | - | 待确认 | [确认成交] |

### 确认成交弹窗

点击"确认成交"后弹出 Modal：

```
确认成交 — 平安银行 (000001)
─────────────────────────────
参考价：12.500    计划股数：7900
实际成交价：[_______]
实际股数：  [_______]  ← 默认填入计划股数
预计金额：  ¥ 0（输入后自动计算）
─────────────────────────────
[跳过（未成交）]          [确认]
```

确认后：
- 写入 actual_price、actual_shares、actual_amount、confirmed_at，status 改为 done
- 若所有 items 均 done/skip，主表 status 改为 done；否则改为 partial
- 页面局部刷新该行状态，不整页刷新

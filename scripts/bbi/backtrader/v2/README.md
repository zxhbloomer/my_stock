# BBI 趋势跟踪策略 v2 — 单股逐一回测（增强版）

## 策略精髓

**v2 在 v1 纯穿越信号基础上，加入了三大增强：动量确认入场、金字塔加仓、ATR 移动止盈 + 多层出场保护。**

核心问题意识：v1 的假穿越太多，持仓 1-14 天的交易胜率仅 7-17%，占总交易量 70%。v2 的目标是过滤假信号、让真实趋势充分展开。

---

## v1 → v2 的核心变化

### 1. 入场：从单一穿越 → 三重确认

| 条件 | v1 | v2 |
|------|----|----|
| BBI 金叉 | ✅ | ✅ |
| BBI 斜率向上 | ❌ | ✅ `BBI[0] > BBI[-3]` |
| MACD 动量确认 | ❌ | ✅ `MACD > Signal 或 MACD > 0` |

三个条件同时满足才入场，大幅减少震荡市假穿越。

### 2. 仓位：从全仓 → 金字塔加仓

| 操作 | v1 | v2 |
|------|----|-----|
| 初始建仓 | 100% 资金 | **50%** 资金 |
| 加仓条件 | 无 | 盈利 ≥ 3% 且 MACD > 0 |
| 加仓量 | — | 剩余 50% 全仓加仓（一次） |

趋势确认后再加仓，降低假突破的成本。

### 3. 出场：从单一死叉 → 三层出场保护

| 层级 | 条件 | 说明 |
|------|------|------|
| 第一层（硬止损） | 亏损 ≥ 8% | 立即出场，不受持仓天数限制 |
| 第二层（持仓保护） | 入场后前 N 日 | 屏蔽软出场信号，给趋势时间展开 |
| 第三层（软出场） | BBI 死叉 / MACD 双死叉 / ATR 移动止盈 | 满足任一即出场 |

### 4. ATR 移动止盈（v2 新增）

```
trail_stop = peak_close - ATR_MULTIPLIER × ATR(14)
trail_stop 只升不降
```

持仓期间动态追踪最高价，锁定趋势利润。

## 回测框架

与 v1 相同，使用 **Backtrader** 框架，多进程逐股回测（`N_WORKERS=8`）。

输出文件：
- `output/stats_summary.csv` — 全市场统计汇总
- `output/trades_detail.csv` — 全部交易明细

## v2 的局限性

v2 对单股趋势跟踪做了充分优化，但本质上仍是**逐股独立回测**：
- 无法模拟真实组合操作（不能同时持有多只股票）
- 全市场 1700+ 只股票同时持有在实盘中不现实
- 缺乏选股排序机制（哪只股票更值得持有？）

这些问题在 v3 中通过引入筹码分布出场信号进一步优化，在 v4_plan 中通过轮动组合模式彻底解决。

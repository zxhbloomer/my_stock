# BBI 趋势跟踪策略 v2

## 策略概述

基于 BBI（Bull and Bear Index，多空指标）的 A 股趋势跟踪策略。使用 Backtrader 框架对全市场股票进行逐一回测，输出统计汇总和交易明细。

---

## 算法与逻辑

### 1. BBI 指标计算

BBI 是四条均线的等权平均，本策略使用 MA(5, 10, 20, 60)（区别于通达信默认的 MA(3,6,12,24)）：

```
BBI = (MA5 + MA10 + MA20 + MA60) / 4
```

MA60 同时作为趋势参考线单独保留。

---

### 2. 入场信号（三个条件同时满足）

| 条件 | 说明 |
|------|------|
| `cross_up` | 前一日收盘价 < 前一日 BBI，当日收盘价 > 当日 BBI（价格从下方穿越 BBI） |
| `bbi_slope` | 当日 BBI > 3 日前 BBI（BBI 本身处于上行趋势） |
| `macd_ok` | MACD 线 > Signal 线，或 MACD 线 > 0（动量确认） |

> 注：早期版本曾加入 `above_ma60`（收盘价 > MA60）条件，经回测验证该条件系统性过滤了底部反转信号，导致胜率和卡玛比下降，已移除。

---

### 3. 仓位管理（金字塔加仓）

- 入场时用可用资金的 **50%** 建仓（`PYRAMID_FIRST_RATIO = 0.5`）
- 持仓盈利达到 **3%** 且 MACD > 0 时，用剩余资金全仓加仓一次（`PYRAMID_ADD_TRIGGER = 0.03`）
- 每次买入数量取整到 100 股（A 股最小交易单位）

---

### 4. 出场逻辑

出场分三层，优先级从高到低：

#### 第一层：硬止损（立即出场，不受持仓天数限制）
```
亏损幅度 = (当前收盘价 - 持仓成本) / 持仓成本 <= -8%
```

#### 第二层：最短持仓保护（入场后 5 日内屏蔽软出场信号）
入场后前 5 个交易日，忽略 BBI 死叉和 ATR 止损信号，避免短期震荡假信号过早打出仓位。

#### 第三层：软出场信号（满足任一即出场）

| 信号 | 条件 |
|------|------|
| BBI 死叉 | 前一日收盘 > 前一日 BBI，当日收盘 < 当日 BBI |
| MACD 双死叉 | MACD 线 < Signal 线 **且** MACD 线 < 0（双重确认，避免假死叉） |
| ATR 移动止盈 | 收盘价 < 历史最高收盘价 - 2.5 × ATR(14)（只升不降的动态止盈线） |

---

### 5. ATR 移动止盈机制

```
peak_close = max(历史收盘价，持仓期间滚动更新)
trail_stop = peak_close - ATR_MULTIPLIER × ATR(14)
trail_stop 只升不降（新计算值 > 当前值才更新）
```

当收盘价跌破 `trail_stop` 时触发出场，锁定趋势行情中的大部分利润。

---

### 6. 交易成本

| 项目 | 参数 |
|------|------|
| 买入佣金 | 0.05%（`COMMISSION_BUY`） |
| 卖出佣金 | 0.15%（含印花税，`COMMISSION_SELL`） |
| 最低佣金 | 5 元（`MIN_COMMISSION`） |
| 初始资金 | 10 万元（`INIT_CASH`） |

---

### 7. 股票筛选条件（`10_prepare_data.py`）

| 条件 | 参数 |
|------|------|
| 上市天数 | ≥ 365 天 |
| 流通市值 | ≥ 100 亿元（`FILTER_MIN_CIRC_MV = 1_000_000` 万元） |
| 日均成交额 | ≥ 5000 万元（`FILTER_MIN_AMOUNT = 50_000` 千元） |

---

## 文件说明

| 文件 | 说明 |
|------|------|
| `config.py` | 所有参数配置 |
| `10_prepare_data.py` | 从 PostgreSQL 拉取数据，计算 BBI/MA60，保存为 parquet |
| `20_run_backtest.py` | 多进程回测主程序，输出 stats_summary.csv 和 trades_detail.csv |
| `30_generate_report.py` | 生成 HTML 可视化报告 |
| `output/stock_data/` | 各股票 parquet 数据文件 |
| `output/stats_summary.csv` | 全市场统计汇总 |
| `output/trades_detail.csv` | 全部交易明细 |

---

## 运行方式

```bash
# 1. 准备数据（需连接 PostgreSQL）
python 10_prepare_data.py

# 2. 运行回测
python 20_run_backtest.py

# 3. 生成报告
python 30_generate_report.py
```

## 版本更新记录

### v2.4 — 2026-04-24 21:30:00

**变更来源：** tmp 分支 v3b 实验验证后合并

**核心变更：出场优化三件套**

| 参数/逻辑 | v2.3 | v2.4 | 说明 |
|-----------|------|------|------|
| `ATR_MULTIPLIER` | 2.5 | **3.5** | 更宽的移动止盈，给趋势更多空间 |
| `MIN_HOLD_DAYS` | 5 | **10** | 延长最短持仓保护期 |
| MACD 死叉出场 | MACD<Signal 且 MACD<0 | **+BBI 也在下行** | 双重确认，避免强趋势中的假死叉 |

**变更原因：**

对 v2.3 的 57,829 笔交易进行持仓天数分析，发现 5-20 天区间（占 64%）胜率仅 7-21%，平均亏损 2-4%。根本原因：ATR×2.5 止损线在趋势初期 ATR 较大时过早触发；MACD 在强趋势中频繁出现短暂死叉假信号。

实验过程：
- v3a（ADX>20 入场过滤）：交易数 -50%，卡玛比 -51%，方向错误——ADX 是滞后指标，A 股 BBI 穿越恰在趋势启动时，此时 ADX 尚低
- v3b（出场优化）：年化 +32%，卡玛比 +13%，胜率 +9.3%，方向正确

**回测效果对比（1708 只股票，2018-01-01 至今）：**

| 指标 | v2.3（改前） | v2.4（改后） | 变化 |
|------|-------------|-------------|------|
| 总交易数 | 57,829 | 51,417 | -11% |
| 平均胜率 | 30.16% | **32.96%** | +2.8pp |
| 平均每笔收益 | 0.05% | **0.27%** | +397% |
| 平均年化收益 | 1.16% | **1.54%** | +32% |
| 平均最大回撤 | 36.89% | 40.16% | +3.3pp |
| 平均卡玛比 | 0.0539 | **0.0611** | +13% |
| 平均持仓天数 | 12.8 天 | 17.4 天 | +4.6 天 |

### v2.3 — 2026-04-24 11:45:00

**变更来源：** tmp 分支实验验证后合并

**核心变更：新增最短持仓保护 + 硬止损**

| 参数 | 值 | 说明 |
|------|----|------|
| `MIN_HOLD_DAYS` | 5 | 入场后前 5 个交易日屏蔽软出场信号 |
| `HARD_STOP_LOSS` | 8% | 亏损超过 8% 立即出场，不受持仓天数限制 |

**变更原因：**

通过对 69K 笔历史交易的持仓天数分析，发现持仓 1-14 天的交易胜率仅 7-17%，占总交易量 70%，是被 MACD 短期震荡或 ATR 在入场初期快速打出的假信号。持仓超过 30 天的交易胜率达 89%，平均盈利 12-41%。加入 5 日保护期后，策略给真实趋势足够时间展开。

**回测效果对比（1708 只股票，2018-01-01 至今）：**

|------|-------------|-------------|------|

| 平均胜率 | 24.41% | **30.23%** | +5.8pp |

### v2.2 — 2026-04-24 10:30:00

**变更来源：** tmp 分支实验验证后合并

**核心变更：移除 `above_ma60` 入场过滤条件**

**变更原因：**

`above_ma60`（收盘价必须在 MA60 上方）系统性地过滤了底部反转信号——BBI 穿越策略的最佳入场点恰恰是价格从 BBI 下方穿越上来时，此时 MA60 往往还在上方。移除后交易数从 31K 增至 69K，卡玛比从 0.0324 升至 0.0401。

| 指标 | v2.1（含 above_ma60） | v2.2（移除后） |

| 总交易数 | 31,405 | 69,223 |
